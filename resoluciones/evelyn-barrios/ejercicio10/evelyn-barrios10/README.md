# Resolución Ejercicio 10 - Lógica de Programación (Autos)

**Autor:** Evelyn Barrios  
**Materia:** Programación con JavaScript

## Descripción
Esta entrega representa un ejercicio de integración de conocimientos aplicados a la gestión de un inventario de automóviles, donde se evalúan condiciones técnicas y comerciales para la toma de decisiones automatizada.

## Justificación Técnica

En la solución se han consolidado los conceptos clave aprendidos en el módulo para resolver un problema de lógica integral:

1.  **Integración de Funciones y Objetos**: Se implementaron funciones que procesan objetos de vehículos para determinar estados complejos, como la necesidad de servicio técnico basado en kilometraje.
2.  **Estructuras de Decisión Complejas**: Se utilizaron operadores lógicos combinados (`&&`, `||`) para validar múltiples criterios simultáneamente (año del modelo, tipo de transmisión y precio de mercado).
3.  **Manejo de Tipos de Datos**: Se demostró el uso correcto de constantes para valores de fábrica y variables para datos que sufren depreciación o cambios de estado.
4.  **Flujo de Control**: Se aplicaron validaciones para evitar errores en el procesamiento de datos vacíos o nulos en el inventario.

## Decisiones de Diseño

*   **Contexto de Concesionaria**: Se simuló un entorno de venta y servicio de autos en Guatemala, utilizando términos como "rodado", "de agencia" y precios en Quetzales para mayor realismo.
*   **Código Modular**: Se priorizó la creación de funciones pequeñas y específicas que permiten evaluar por separado la seguridad, el precio y el estado mecánico del vehículo.
*   **Formateo de Salida**: Se utilizaron *Template Literals* para generar reportes de inspección claros y profesionales en la consola.

Este ejercicio sirve como cierre de los conceptos fundamentales, demostrando la capacidad de orquestar diferentes herramientas de JavaScript en la resolución de un problema de lógica de negocio completo.
